El Problema

Sin datos en tiempo real, cada falla es una sorpresa costosa

MetalTech Argentina tiene 18 máquinas con sensores activos pero ningún sistema central que los integre. Los datos terminan en planillas Excel.

MetalTech Argentina S.A. es una empresa metalúrgica mediana del Gran Buenos Aires con 50 empleados y 3 líneas de producción. La infraestructura de sensores ya está instalada — el problema es la falta de un sistema que integre, analice y alerte sobre los datos en tiempo real.

La Solución

Un sistema que convierte datos en decisiones preventivas

SensorGuard integra todos los sensores existentes vía API REST, centraliza la información en un dashboard en tiempo real y genera alertas automáticas antes de que ocurra una falla.

Análisis Estratégico

FODA del proyecto

Evaluación de factores internos y externos que impactan el desarrollo y adopción de SensorGuard.

INTERNO EXTERNO
Fortalezas
Interno · Positivo
Capacidades propias del equipo
  • Equipo con dominio de metodología Scrum y Kanban
  • Conocimiento sólido de IoT e Industria 4.0
  • Rol Analista/Tester integrado desde el inicio del proyecto
  • Comunicación fluida y roles claramente definidos entre integrantes
Oportunidades
Externo · Positivo
Factores del entorno favorables
  • Creciente adopción del IIoT en la manufactura argentina
  • Sensores IoT de temperatura y vibración cada vez más accesibles en costo
  • Alta demanda de sistemas de mantenimiento predictivo en PyMEs industriales
  • El cliente ya tiene sensores instalados, reduciendo la inversión inicial
Debilidades
Interno · Negativo
Limitaciones del equipo o proyecto
  • Primera versión acotada (MVP), no un sistema completo de mantenimiento
  • Sin experiencia previa trabajando con sistemas industriales reales
  • Dependencia de la infraestructura de sensores del cliente (no gestionada por el equipo)
  • Tiempo de desarrollo limitado para el primer ciclo (sprints académicos)
Amenazas
Externo · Negativo
Riesgos del entorno
  • SAP PM e IBM Maximo ya dominan el segmento enterprise del mercado
  • Resistencia cultural al cambio por parte del personal operativo
  • Vulnerabilidades de ciberseguridad inherentes a sistemas IoT conectados
  • Variaciones presupuestarias del cliente pueden afectar el alcance
Requerimientos Funcionales

Qué hace el sistema

7 requerimientos funcionales que conforman el MVP. Cada uno resuelve un problema concreto de MetalTech.

RF01
📊
Dashboard en tiempo real
Visualización del estado de temperatura y vibración de cada máquina. Los datos se actualizan automáticamente sin necesidad de recargar la página.
⏱ Actualización cada 10 segundos
RF02
🔔
Alertas automáticas por email
Cuando un sensor supera el umbral configurado, el sistema envía automáticamente un email al supervisor sin intervención manual.
⚡ Notificación en menos de 30 segundos
RF03
🔧
Órdenes de mantenimiento
Flujo digital completo para gestionar el trabajo de mantenimiento: creación de la orden, asignación al técnico responsable, seguimiento y cierre documentado.
📋 Crear · Asignar · Cerrar
RF04
📈
Historial y trazabilidad
Registro completo de lecturas de sensores y fallas por máquina, consultable filtrando por rango de fechas para análisis histórico y auditoría.
🗓 Consulta por rango de fechas
RF05
👥
Perfiles y roles de usuario
Tres niveles de acceso adaptados a la estructura de MetalTech: Operario ve solo el dashboard; Supervisor gestiona alertas y órdenes; Gerente accede a reportes y configuración.
👷 Operario · 👔 Supervisor · 📊 Gerente
RF06
⚙️
Configuración de umbrales
Cada máquina puede tener umbrales de temperatura y vibración configurados de forma independiente, adaptándose a las especificaciones de cada equipo industrial.
🎛 Por máquina, configurable desde la app
RF07
📄
Exportación de reportes PDF
Generación de reportes en formato PDF con el historial de fallas, órdenes completadas y métricas de disponibilidad para presentar a dirección o clientes.
📥 Formato PDF descargable
Arquitectura y Requerimientos No Funcionales

Stack técnico

Flujo de datos desde el sensor hasta el usuario, con las restricciones de calidad que garantizan un sistema confiable para producción industrial.

📡
Capa de sensores IoT
Sensores de temperatura y vibración ya instalados en las 18 máquinas de MetalTech. Envían datos vía HTTP a la API del sistema.
↓ API REST (HTTP)
⚙️
Backend + Lógica de negocio
Valida lecturas, compara contra umbrales configurados, dispara alertas por email y persiste datos. Expone la API para el frontend.
↓ ORM / SQL
🗄️
Base de datos relacional
Almacena lecturas históricas, umbrales por máquina, órdenes de mantenimiento, usuarios y roles. Cifrado AES-256 en reposo.
↓ HTTPS

Frontend web (responsive)
Dashboard accesible desde tablet y desktop. Sin app móvil nativa en la versión MVP. Interfaz diferenciada por rol de usuario.
